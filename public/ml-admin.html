Config Notes

Self-play games per cycle controls how many fresh games are generated before the trainer samples from the replay buffer.
Belief samples per move controls how broadly shared-tree ISMCTS samples hidden worlds that fit the public history. It no longer spins up a separate MCTS tree per identity hypothesis.
Backend selection, device preference, game worker concurrency, worker refresh cadence, graph stride, and risk bias are auto-managed.
Promotion is based on head-to-head results, not just loss. A new checkpoint only becomes the best generation if it reaches the configured win-rate threshold against the current best generation and the earlier promoted generations you include.
Runs stop when they hit the failed-promotion cap, which is the plateau guard for longer sweeps.
Replay data is retained for recent run games so you can inspect generation matchups later without rebuilding the run.

How It Works

This tab no longer launches one-off games directly.
Each checked model becomes a normal bot option in the main Cloak's Gambit client, using the same live routes and standard match flow as the built-in bots.
When a player chooses one of those enabled promoted models, the ML runtime auto-setups, auto-readies, and plays the bot seat with that generation's Node inference stack.
Uncheck a model here and save to remove it from the public bot dropdown without deleting the underlying run or generation.
